32. Perbedaan sip request dan sip server


SIP server merupakan aplikasi yang bertujuan untuk menerima. Sementara itu, SIP request merupakan aplikasi yang bertujuan untuk menjawab.

Pembahasan

SIP merupakan singkatan dari Session Initiation Protocol. SIP sendiri merupakan suatu persinyalan protokol dan pengontrolan lapisan atau layer aplikasi. Fungsi dari SIP adalah untuk membangun dan mengakhiri sebuah sesi multimedia.

Sesi multimedia merupakan sebutan bagi sesi di mana pengguna suatu aplikasi saling bertukar data. Data tersebut dapat berupa teks, suara, maupun video. Di situ, tugas SIP adalah untuk menyediakan dasar bagi protokol aplikasi lain.

Berikut ini adalah fungsi 4 server SIP secara singkat:

Proxy server, yaitu server yang berfungsi sebagai server dan client. Server ini juga melakukan routing, memastikan request untuk pihak yang berizin, dan membuat kebijakan.Redirect server, yaitu server yang berfungsi untuk menyelesaikan tugas request yang diberikan pengguna aplikasi.Registrar server, yaitu server yang berfungsi unutk menerima dan melakukan proses pada pemberitahuan pendaftaran dengan cara mengizinkan akses lokasi keberadaan suatu endpoint.Location server, yaitu server yang berfungsi untuk menyediakan layanan pada database abstrak dengan tujuan menerjemahkan keterangan pada domain jaringan.
